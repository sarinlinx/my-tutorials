ESLint

This configures ESLint to work with Prettier.

Install dev dependencies

eslint-config-prettier disables formatting in ESLint.

eslint-plugin-prettier allows ESLint to show formatting erros while you type.

eslint-config-airbnb sets a JavaScript style guide to follow.

eslint-pluging-node adds ESLint rules for Node.js.

These 3 allow the style guide to work. eslint-plugin-import eslint-plugin-jsx-ally eslint-plugin-react.

npm i eslint prettier eslint-config-prettier eslint-plugin-prettier eslint-config-airbnb eslint-plugin-node eslint-plugin-import eslint-plugin-jsx-ally eslint-plugin-react

Create a config file

Create a file named .eslintrc.json in your application directory.

This is the same location as your .prettierrc file.

This allows you to adjust the rules ESLint uses. For example:

  • no-console: ESLint displays an error when you use console.log. This turns it into a warning.
  • consistent-return: Every function should always return something. This is turned off.
  • no-unused-vars: This catches unused variables, but adds a few exceptions.
{
  "extends": ["airbnb", "prettier", "plugin:node/recommended"],
  "plugins": ["prettier"],
  "rules": {
    "prettier/prettier": "error",
    "spaced-comment": "off",
    "no-console": "warn",
    "consistent-return": "off",
    "func-names": "off",
    "object-shorthand": "off",
    "no-process-exit": "off",
    "no-param-reassign": "off",
    "no-return-await": "off",
    "no-underscore-dangle": "off",
    "class-methods-use-this": "off",
    "prefer-destructuring": ["error", { "object": true, "array": false }],
    "no-unused-vars": ["error", { "argsIgnorePattern": "req|res|next|val" }]
  }
}
