Events

Events are something that has taken place on the web page.

Events can represent everything from basic user interactions to automated notifications of things happening in the rendering model.

The Events Queue

Events in JavaScript work by inserting themselves into a queue and they will remain there until they're needed. So if you don't remove some events, it's going to cause a problem like this.

Uncaught TypeError: Cannot read property 'parentNode' of null.

Different types of events

Browser level events

Load - when the resource and dependents have finished loading

Error - the resource failed to load

Online/offline -

Resize - when viewport is resized

Scroll - when viewport is scrolled

DOM level events

Focus - When element receives focus (clicked, tabbed to, etc.)

Resets/submit - form-specific events

Mouse events - click, mousover, drag, drop, etc.

Other events

Media events - relate to audio/video playback

Progress events - monitor ongoing browser progress

CSS transition events - transition start/run/end

Event Registration

Event registraton is when you tell the browser you want to do something when an event takes place.

There are 3 ways to do this.

Option 1 - Using tag attribute

This create an event handler for your event using an onclick tag.

This is bad practice as it makes your code difficult to update.

<button onclick="alert('pushed button')">

Option 2 - Dot notation (old option)

First specify element.

Then get the element by ID or Class.

Give it an event handler. In this example, onclick().

The event handler then executes a function literal (function without a name)

<img src"pink.gif" "alt="pink" id="pink">

document.getElementById('pink').onclick=function() {
  alert('click on pink");
}

Option 3 - Using addEventListener() (Modern option)

First specify element.

Add 'addEventListener'.

Pass along type of event you want (click).

There are a few advantages to this:

  • This allows you to check for multiple events within a single call aka: Event propagation.
  • The other less common advantage is that events can be triggered by non-dom elements, which means that with that method you can write your own event.
  • And different parts of your code can listen for those events that you've created yourself, and then you can execute pieces of code based on that.

NOTE: true or false dictates 'capturing phase' or 'bubling phase'.

<img src"pink.gif" "alt="pink" id="pink">

document.getElementById('pink').addEventListener('click', function() {
  alert("click on pink");
} , false);

You can also write the function separate from the addEventListener call.

NOTE How there are no () after the reveal function.

<img src"pink.gif" class="cta">

const CTA = document.querySelector(".cta a");

function reveal(e) {
  CTA.classList.toggle("hide");
}

CTA.addEventListener("click" , reveal, false);

Example - changing the text of a button

When the button is clicked, the text changes.

<button>Create Note</button>

document.querySelector('button').addEventListener('click', function (e) {
  //Take the Object 'e', then assign its property of textContent to a new string
  e.target.textContent = 'The button was clicked'
})
