Namespacing

Namespacing allows you to protect any variables that you have in your modules from any global scoped variables.

This is really important because in any module, you might be using some variables that are already used by other things in that application.

Encapsulating a function within parentheses protects all the variables inside from the global scope.

But what if you need to access those variables from within the application? To do that, you need to create a variable and assign it to your self-executing function. This gives you access to what's inside it.
