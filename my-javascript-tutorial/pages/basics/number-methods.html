Overview

With JavaScript, methods and properties are available to primitive values, because JavaScript treats primitive values as objects when executing methods and properties.

Primitive number values would be: 3.14 or 2014.

toString()

toString() returns a number as a string.

Why does the typeof return 'number'?

var x = 123;
x.toString();
console.log(typeof x);

valueOf()

The valueOf() method is used internally in JavaScript to convert Number objects to primitive values.

There is no reason to use it in your code.

All JavaScript data types have a valueOf() and a toString() method.

toFixed()

toFixed() returns a string, with the number written with a specified number of decimals

This returns:

  • 10
  • 9.66
  • 6.6560
var x = 9.656;
x.toFixed(0);
x.toFixed(2);
x.toFixed(4);

toPrecision()

toPrecision() returns a string, with a number written with a specified length:

This returns:

  • 9.656
  • 9.7
  • 9.656
  • 9.65600
var x = 9.656;
x.toPrecision();

Math methods

round()

round() rounds off the value.

This returns:

  • 104
let num = 103.941
console.log(Math.round(num))

ceil()

ceil() is ceiling method. It rounds UP to the nearest integer.

So even though the decimal is .2, this rounds UP to 104.

let num = 103.2
console.log(Math.ceil(num))

floor()

rounds down to the nearest integer.

So even though the decimal is .8, this rounds DOWN to 103.

let num = 103.8
console.log(Math.floor(num))

random()

This generates a value between 0 and under 1 (.99999), which you can then scale to your desired range.

This returns a number such as: 0.08616273575050071

let randomNum = Math.random()
console.log(randomNum)

Generate random number between 0 and 10

If you mutiply Math.random() by 10, the highest value will be 9.9999.

To obtain the value 10, multipley by 11 instead. Now the top value could be 10.9999.

Finally, use Math.floor() to round that down to 10.

let randValue = Math.floor(Math.random() * 11);
console.log(randValue)

Generate random number between two specific values

This generates a value between 10 and 20.

You need to use a simple formula of (max - min +1) to ge a base over the min value.

You can then use Math.floor() to round down to the nearest integer to set the base value.

let min = 10
let max = 20
//random() creates a value between 0 and 1
//To get whole numbers between 10 and 20 use this formula: * (max-min)
//The max is now 9.99999 because .9999 * 10. But you only want values from 10 - 20
//So, add +1 to make the base value over 10, now the base value can be 10.999
//Finally, use Math.floor to round down to the nearest integer. This makes the random base value 10
let randomNum = Math.floor(Math.random() * (max - min + 1)) + min
console.log(randomNum)

Another way to write this

This generates a value between 15 and 27.

function getRndInteger(min, max) {
  return Math.floor(Math.random() * (max - min + 1) ) + min;
}

let randomValue = getRndInteger(15,27)
console.log(randomValue)

Number Properties

Returns the largest number possible in JavaScript

This returns: 1.7976931348623157e+308

Returns the smallest number possible in JavaScript

This returns: 5e-324

var x = Number.MIN_VALUE;

Returns the smallest number possible in JavaScript
