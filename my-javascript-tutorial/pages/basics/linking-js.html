3 ways to link JS to you page

There are 3 ways to link JS to your page.

  • External style sheet (Recommended)
  • Internal style sheet
  • Inline

JS is rendered as the browser becomes aware of it. So order matters.

External style sheet (Recommended)

You should link this in the <head> or after </body> depending on when the code needs to be ran.

Placing JavaScripts in external files has some advantages:

  • It separates HTML and code
  • It makes HTML and JavaScript easier to read and maintain
  • Cached JavaScript files can speed up page loads

This links in the <head> section of the HTML page.

<head>
  <script src="myScript.js"></script>
</head>

HTTP/2

HTTP/2 allows multiple assets to be downloaded in parallel. However, if the file is referenced in the header, render blocking will still happen, it just takes less time.

Historically, it was a good idea to place scripts at the bottom of the <body> element to help page load. But since HTTP/2, that is no longer necessary. Today, you can choose to load the JS file in 3 ways:

  • Immediately - If we load it right away, the HTML will parse until it encounters a reference to JavaScript. Then the JavaScript file is loaded, executed, and we continue the HTML parsing. This is JavaScript render blocking.
  • Asynchronously - If we load our JavaScript file asynchronously by adding the async attribute, the file will be downloaded alongside the HTML. Render blocking only kicks in when the script is executed. It's better, and useful in some situations.
  • Deferred - <script src="scrip.js" defer></script> - If we defer the script, loading and execution of the file, even if it's referenced in the head, only happens when everything else is loaded.

Internal style sheet

Adding code in the <head> section.

Add your code to the head section if you want the JS to run before page is rendered.

For example, if you add document.body.innerHTML it will throw an error because it's BEFORE the <body>.

<html>
  <head>
    <script>
      code...
    </script>
  </head>

    <body>
    </body>
</html>

Adding code after the <body> tag.

If you want JS to run after page has loaded.

If your JS runs on something in the <body>, it must be after the body tag or deferred to render later.

<html>
  <head>
  </head>

  <body>
  </body>

    <script>
      code...
    </script>

</html>

JS render blocking

Because JavaScript may change HTML and CSS on a page, any time a browser encounters a JavaScript file reference, it stops the rendering of the page until the script file is downloaded and executed.

This is known as JavaScript render blocking and it can slow down the performance of a site significantly.
