Default TRUE (truthy) values

The following values are always truthy:

  • '0' (a string containing a single zero)
  • 'false' (a string containing the text “false”)
  • [] (an empty array)
  • {} (an empty object)
  • function(){} (an “empty” function)

Default FALSE (falsy) values

The following values are always falsy:

  • false
  • 0 (zero)
  • '' or "" (empty string)
  • null
  • undefined
  • NaN (e.g. the result of 1/0)

AND

AND returns last TRUE value in a series of TRUE

// Left operand is true,
// AND needs to check the right operand also
// It then returns that last TRUE value

alert(1 && 0) // 0, the last evaluated value
alert(1 && 5) // 5, the last evaluated value

AND returns FALSE if the first value is FALSE.

// Left operand is false,
// AND stops immediately and returns it

alert(null && 5) // Since the first value if FALSE (null), it is the last evaluated value

Logical OR with falsy operands

When using OR with falsy operands, it looks for the TRUE value.

In this example, the left operand is TRUE, so OR stops and returns it immediately.

The result is 1.

alert(1 || 0)

In this example, the left operand is FALSE, so OR continues to the next operand which is TRUE and returns it.

The result is 1.

alert(0 || 1)

If both are false

In this example, the left operand is FALSE, so OR continues to the next operand which is also FALSE.

Since there is no TRUE value, the result is null.

alert(0 || 1)

When to use?

This is useful when you have multiple variables and need to get the first one that is true.

This returns 'Oh-la-la' since it's the last value that isn't falsy.

var b = false
var n = 0
var s = "O-la-la"

var result = b || n || s

alert(result) // "O-la-la"

NOT

NOT converts its operand into boolean, then negates it.

alert( !true ) // returns false
alert( !0 ) // returns true

Double NOT

A Double NOT converts its value into boolean.

alert( !!"string" ) //	returns true
alert( !!null ) // returns false

Boolean conversion

Traditionally, it is possible to put a value into if. In this case, it gets converted into boolean.

var x = 1

if (x) {
  alert('true')
}

Short Circuiting with ||

Logical operators can do 3 things

Use any data type

Return any data type

short-cuiting (short-circuit evaluation)

Short-circuiting example

OR looks for true values to confirm if the statement is true.

This means short-circuting with OR looks for true values to return.

It returns the first truthy value of all operands, or the last value if all are falsey

You can use this to set default values.

truthy values (1)

This prints 3

If the first value is a truthy value it immediately returns this value

The 2nd value isn't even looked at.

console.log(3 || 'Me')

truthy values (2)

This prints true

If the first value is a truthy value it immediately returns this value

The 2nd value isn't even looked at.

console.log(true || 0)

falsey values

This prints Me

If the first value is a falsey value, the 2nd value is evaluated and returned.

console.log('' || 'Me')

falsey values

This prints null since undefined is a falesy value.

console.log(undefined || null)

falsey values

This prints Hello because the first 3 values are falesy.

Hello is the first truthy value.

console.log(undefined || 0 || '' || 'Hello' || 23 || null)

Setting a default value if the value doesn't exist

This checks if restaurant.numGuests exists. If so, the number is returned. If not, the default value of is returned.

This can be used instead of the ternery operator.

NOTE: This will not work if guests value is zero since that's a value. Look at nullish operator for solution.

//ternery example
const guests1 = restaurant.numGuests ? restaurant.numGuests : 10;
console.log(guest1)

// short circuiting example
const guests2 = restaurant.numGuests || 10;
console.log(guests2)

Short Circuiting with &

& short-circuits when the first value is falsey.

This is the opposite of ||

If the first value is false, the entire operation is false

Returns first falesy value or all if they are all truthy

You can use it to execute code in the 2nd operand if the first is true.

falesy example

This returns 0

console.log(0 && 'me')

truthy example

This returns me since the last operand is true.

console.log(7 && 'me')

example

This returns null since it's the first falsey value.

The operation short-circuits and stops there.

console.log('Hello' && 7 && null && 'me')

replacing if statement

You can use the && operator to replace the if statement

This says that if the first statement (restaurant.orderPizza) doesn't exist, the evaluation is short-circuited and nothing happens.

If it does exist it is truthy and the next operand is evaluated and returned.

// if statement
if (restaurant.orderPizza) {
  restaurant.orderPizza('mushrooms', 'spinach');
}
                  
// && example
restaurant.orderPizza && restaurant.orderPizz('mushrooms', 'spinach')
