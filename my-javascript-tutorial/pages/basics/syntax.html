Case

Case matters. JavaScript uses camelCase as a standard naming convention. For example: getElementByTagName.

Var, function, method names should start with lowercase.

Object and class names start with Capital.

Constants are all caps.

White space is ignored.

End each statement with a semicolon;

Tabs are ignored.

No hyphens Hyphens are not allowed in JavaScript. It is reserved for subtractions.

Data Types

Numeric - For exampe: var num = 1.687;

String - For exampe: var string = "string text here";

  • Single or double quotes are ok. But, if quotes are in the string, use single quotes to encase it: var str = 'If he says "Hello" make sure to respond'
  • or use double quotes, but escape them in the string: var str = "If he says \"Hello\" make sure to respond"

Boolean - For exampe: var bool = true;

null - These are empty, but not undefined. For exampe: var asdf = null;

undefined - These are not set to anything. They are just placeholders. For exampe: var stuff;

Symbol (ES6)

Viewing the type of variable

Use 'typeof' to determine what type the variable is. For example: console.log(typeof varNameHere);

Newlines

Newlines are NOT ignored. A newline may separate statements, just like semicolons do. For example, these two are equivalent:

a = 5
a = 5;

If necessary, you can insert a convenience newline with \. A backslash before a line break forces the interpreter to ignore the newline. For example:

var a = "long \
line "

The newline is also ignored if the expression is not finished.

Concatenate

The + operator concats info.

var a = "my" + "string here" //returns my string here

It doesn't matter which one is a string. For example.

alert( '1' + 2) // returns 12
