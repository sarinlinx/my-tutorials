String comparison

Strings are compared in phone book alpha order.

This results in TRUE since Anna is alphabetically after Abba.

alert( "Anna" > "Abba" )

This is also case sensitive. This result is TRUE

alter( "abba" > "Zend" )

Comparing different types

Number comparison

This returns TRUE becuase 8 is less than or equal to 10.

console.log(8 <= 10)

The interpreter converts values to numbers. In this case, "01" becomes the numerical value of 1.

This results in TRUE.

alert( "01" < 2 );

This also results in TRUE because true converts to 1 in boolean.

alert(1 == true);

NOTE: Be careful with string numbers since it will go in alpha numeric order.

This results is TRUE since 2 is > than 14 when using lexigraphical order (dictionary sorting).

This is because lexicographical order puts everything in alpha numberic order. So ANY number that start with a 1 comes before ANY number that starts with a 2.

alert ( "2" > "14" );

Uncomparable

undefined values can't be compared with numbers.

undefined > 0 // is false
undefined == 0 // is false

Comparison Equality operators

==

Two equal signs is a general equality operator.

This result is TRUE.

alert(0 == false);

Strict equality operator ===

Three equal signs is a strict equality operator. The values must match exactly.

This result is FALSE since values of different type are always unequal.

alert(0 === false);

In-equality operator !=

This is false since zero does equal a false boolean value. The type doesn't matter.

console.log(0 != false)

Strict in-equality operator !==

This is true since zero is not equal to false since they are different types.

console.log(0 !== false);

Special values

This is weird and something to remember to look out for.

null > 0 is false

null == 0 is false

but...

null >= 0 is true
