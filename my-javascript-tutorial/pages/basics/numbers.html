Overview

JavaScript has only one type of number. Numbers can be written with or without decimals.

The maximum number of decimals is 17, but floating point arithmetic is not always 100% accurate.

Never write a number with a leading zero (like 07). Some JavaScript versions interpret numbers as octal if they are written with a leading zero.

Numbers can be written with or without decimals.

var x = 3.14;
var y = 3;

Extra large or extra small numbers can be written with scientific (exponent) notation.

var x = 123e5;
var y = 123e-5;

Number types

Unlike other programming languages, JavaScript does not define different types of numbers, like integers, short, long, floating-point etc.

JavaScript numbers are always stored as double precision floating point numbers, following the international IEEE 754 standard.

JavaScript Numbers are Always 64-bit Floating Point

Adding numbers and strings

The + sing adds for numbers and concatenates for strings.

If you use + with numbers and strings, the result will be a string concatenation.

This results in 1020

var x = 10;
var y = "20";
var z = x + y;

Order matters!

The JavaScript compiler works from left to right. So take that into account when several values use the + sign.

The result depends on the order from left to right of 'string' or 'number'.

This results in 1020

Why? Because 'z' starts as a string, the + sign is then used to concatenate.

var x = 10;
var y = 20;
var z = "The result is: " + x + y;

This results in 3030

Why? Because x and y are first added. Then, the string 'z' is concatenated.

var x = 10;
var y = 20;
var z = "30";
var result = x + y + z;

Numeric Strings

JavaScript will try to convert strings to numbers in all numeric operations.

You can use - * or /

You CANNOT use + since that will concatenate the strings.

This results in 10

var x = "100";
var y = "10";
var z = x / y;

NaN - Not a Number

NaN is a JavaScript reserved word indicating that a number is not a legal number.

Trying to do arithmetic with a non-numeric string will result in NaN (Not a Number)

This results with: NaN

var x = "100";
  var y = "10";
  var z = x / y;

What if the string contains a numeric value?

In this case, JavaScript will consider the string to be a number and divide.

var x = 100 / "10";

Using NaN in a math operation with a number

If NaN is used in math, it will return NaN.

var x = NaN;
var y = 5;
var z = x + y;

Using NaN in a math operation with a string

If NaN is used with a string and the + sign, it concatenates

This returns: NaN5

var y = "5";
var z = x + y;

Check if a value is a number

You can use the global JavaScript function isNaN() to find out if a value is a number.

isNan() returns true or false

NaN is a number type

NaN is actually a number type in JavaScript.

You can check by using typeof

alert(typeof x);

The Infinity value

Infinity is a number: typeof Infinity returns number.

Calculating values too high

If the value returned is larger than the 'largest possible number' (which is 1.7976931348623157e+308), JavaScript returns 'Infinity'.

This will start to calculate values until it's too high, then 'Infinity' is returned.

var txt = "";
while (myNumber != Infinity) {
   myNumber = myNumber * myNumber;

}
console.log(myNumber);

Dividing by zero

Division by 0 (zero) also generates Infinity

var x =  2 / 0;

Hexidecimal values

Calculating values too high

If you precede a value with 0x, JavaScript interprets it as a Hexidecimal value.

This returns 255

var x = 0xFF;

Base values

Calculating values too high

By default, JavaScript displays numbers as base 10 decimals. But you can use the toString() method to output numbers from base 2 to base 36.

var myNumber = 32;
myNumber.toString(10);  // returns 32
myNumber.toString(32);  // returns 10
myNumber.toString(16);  // returns 20
myNumber.toString(8);   // returns 40
myNumber.toString(2);   // returns 100000
