Creating Objects

Objects store data such as properties and methods.

  • Properties: Info about the object we can retrieve and use
  • Methods: functions that belong to the object

Object conversion

Avoid String, Number, and Boolean objects. They complicate your code and slow down execution speed.

var x = new String(); // Declares x as a String object
var y = new Number(); // Declares y as a Number object
var z = new Boolean(); // Declares z as a Boolean object

Objects in JavaScript can be converted to primitives in three contexts:

  • Numeric
  • String
  • Boolean

Comparing Objects to each other using ===

Comparing two objects with === does not work. This is because even if the Objects have the exact same properties and values, they are stored in memory in different locations.

What makes them equal is if they are the same object stored in memory.

For example, this will never work.

console.log( {} === {})

This will work since they are referecing the same Object in memory.

let someObject = {}let otherObject = someObject
console.log(someObject === otherObject)

Enhanced Object literals (ES6)

ES6 introduced 3 new ways to simplify the creation of Object literals.

Old way of adding an Object into another Object

The old way is to create a new property in the Object and assign the variable name of the 2nd Object.

const openingHours = {
  [weekdays[3]]: {
    open: 12,
    close: 22,
  },
  [weekdays[4]]: {
    open: 11,
    close: 23,
  },
  [weekdays[5]]: {
    open: 0, // Open 24 hours
    close: 24,
  },
};

const restaurant = {
  name: 'Classico Italiano',
  location: 'Via Angelo Tavanti 23, Firenze, Italy',
  categories: ['Italian', 'Pizzeria', 'Vegetarian', 'Organic'],
  starterMenu: ['Focaccia', 'Bruschetta', 'Garlic Bread', 'Caprese Salad'],
  mainMenu: ['Pizza', 'Pasta', 'Risotto'],
  openingHours: openingHours,
}

New way in ES6 to call another Object

Just add the variable name of the 2nd Object in to the 1st Object.

const openingHours = {
  [weekdays[3]]: {
    open: 12,
    close: 22,
  },
  [weekdays[4]]: {
    open: 11,
    close: 23,
  },
  [weekdays[5]]: {
    open: 0, // Open 24 hours
    close: 24,
  },
};

const restaurant = {
  name: 'Classico Italiano',
  location: 'Via Angelo Tavanti 23, Firenze, Italy',
  categories: ['Italian', 'Pizzeria', 'Vegetarian', 'Organic'],
  starterMenu: ['Focaccia', 'Bruschetta', 'Garlic Bread', 'Caprese Salad'],
  mainMenu: ['Pizza', 'Pasta', 'Risotto'],
  openingHours,
}

New way in ES6 to write a method

In ES6, you do not have to set a property and write a function expression.

You can remove the the :function part now to simplify the code.

// old way
order: function (starterIndex, mainIndex) {
  return [this.starterMenu[starterIndex], this.mainMenu[mainIndex]];
},

// ES6 syntax
order(starterIndex, mainIndex) {
  return [this.starterMenu[starterIndex], this.mainMenu[mainIndex]];
},

You can compute property names instead of writing them out manually and literally.

This shows the old and ES6 way to extract property names from an Array.

The old way, you have to manually type out the property names.

The ES6 way, you can call the index of the Array, or even compute the property name to something else.

// old way
  const weekdays = ['mon', 'tue', 'wed', 'thu', 'fri', 'sat', 'sun'];

  const openingHours = {
    thu: {
      open: 12,
      close: 22,
    },
    fri: {
      open: 11,
      close: 23,
    },
    sat: {
      open: 0, // Open 24 hours
      close: 24,
    },
  }; 
  
// ES6 syntax
const weekdays = ['mon', 'tue', 'wed', 'thu', 'fri', 'sat', 'sun'];

const openingHours = {
  [weekdays[3]]: {
    open: 12,
    close: 22,
  },
  [weekdays[4]]: {
    open: 11,
    close: 23,
  },
  [`day-${2+4}`]: {
    open: 0, // Open 24 hours
    close: 24,
  },
};

Optional Chaining

If a certain property does not exist, undefined is immediately returned.

Example

The Optional Chaining value is a question mark followed by a period.

?.

If the property before the ? exists, the next property will be read. If it doesn't exist, undefined is returned.

For example

// old way to check if property exists
  
  if (restaurant.openingHours && restaurant.openingHours.mon)
console.log(restaurant.openingHours.mon.open)


// Optional Chaining 
console.log(restaurant.openingHours.mon?.open)

Multiple Optional Chaining

This first checks if openingHours exists. If so, the new property .mon is checked. If that exists, the next is checked.

console.log(restaurant.openingHours?.mon?.open)

Looping through Array while Chaining

This checks what days a restaurant is open.

It firsts loops through the 'days' Array to retrieve the day name.

The variable 'open' is created to check if restaurant.openingHours[].open exists for the specific day during the iteration of the loop. This is fromthe imported Object 'openingHours'.

If the .open property exists on the day being iterated, the hours are printed. The nullish operator is used for the value of zero.

If the .open property does not exists, the default value of 'closed' is printed.

const weekdays = ['mon', 'tue', 'wed', 'thu', 'fri', 'sat', 'sun'];

const openingHours = {
  [weekdays[3]]: {
    open: 12,
    close: 22,
  },
  [weekdays[4]]: {
    open: 11,
    close: 23,
  },
  [weekdays[5]]: {
    open: 0, // Open 24 hours
    close: 24,
  },
};

const restaurant = {
  openingHours,
};


const days = ['mon', 'tue', 'wed', 'thu', 'fri', 'sat', 'sun'];

// Loops through days Array
for (const day of days) {
  // check specific index to see if openingHours.open exists
// nullish operator ?? is needed for the value of zero
const open = restaurant.openingHours[day]?.open ?? 'closed';
console.log(`On ${day}, we open at ${open}`);
}

Optional Chaining with Methods

Checks if a method exists before you call it.

If so it's called.

If not, the default value is printed.


  const restaurant = {
    name: 'Classico Italiano',
    location: 'Via Angelo Tavanti 23, Firenze, Italy',
    categories: ['Italian', 'Pizzeria', 'Vegetarian', 'Organic'],
    starterMenu: ['Focaccia', 'Bruschetta', 'Garlic Bread', 'Caprese Salad'],
    mainMenu: ['Pizza', 'Pasta', 'Risotto'],
  
    // ES6 enhanced object literals
    openingHours,
  
    order(starterIndex, mainIndex) {
      return [this.starterMenu[starterIndex], this.mainMenu[mainIndex]];
    },
}

console.log(restaurant.order?.(0,1) ?? "Method Does NOT Exist")
