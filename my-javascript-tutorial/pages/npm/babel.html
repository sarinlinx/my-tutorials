Babel

Babel transpiles modern code back to ES5 code.

This is necessary for very old browsers.

Converting everything using a Preset

Presets will automatically select which JavaScript features should be compiled based on browser support.

By default, Parcel uses the @babel/preset-env

NOTE: The presetn only includes final features part of the language afer passing the 4 stages of the ECMA process.

Babel can actually only transpile ES6 Syntax. So data things like arrow functions, classes, const, or the spread operator. So these are basically things that have an equivalent way of writing them in ES5.

Polyfilling

New features like find() and promise() cannot be transpiled.

But you can polyfill them.

You can import everything using core-js/stable. This is most common.

But you could also only import only a single part of the core-js library.

Polyfilling will not replace the existing new code. It will recreate them somewhere else in the code.

npm i core-js


import 'core-js/stable/promise';

Install regenerator-runtime

This is also needed for polyfilling facing functions.

npm install regenerator-runtime


import 'regenerator-runtime/runtime';
