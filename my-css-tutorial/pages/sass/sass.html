Sass

Sass is a CSS Preprocessor.

You write sass code in a sass file. Then compile it. This compiler converts it into regular CSS code.

Sass reduces repetition of CSS and therefore saves time.

Sass files has the ".scss" file extension.

Sass supports standard CSS comments /* comment */, and in addition it supports inline comments // comment:

Sass uses variables.

You can next selectors inside of each other, allowing you to write less code.

Operators can be used.

Partials and imports. Allows you to write CSS in different files and import them into a single file.

Mixins: Reusable pieces of CSS code.

Functions. Produce a value that can be used.

Extends. Make different selectors inherit declarations that are common to all of them.

Control directives: Uses conditionals and loops. Not often used.

Variables

Overview

Variables allow you to store data that you would otherwise write out.

With Sass, you can store information in variables, like:

  • strings
  • numbers
  • colors
  • booleans
  • lists
  • nulls

Sass uses the $ symbol, followed by a name, to declare variables:

Ex 1:

This stores the background-color in a variable. You can then use that variable in the CSS intead of the hex code.

//Sass variable
$width = 30px;
$color-primary: #f9ed69;

nav {
  margin: $width;
  background-color: $color-primary;
}

Scope

Add the variables at the top of the script.

If you put them in a tag (like an h1), they will only be available within that tag.

Ex 1:

The output of the paragraph is red since the variable was declared in another tag.

$myColor: red;

h1 {
  $myColor: green;
  color: $myColor;
}

p {
  color: $myColor;
}

!global

Use !global to create a global variable. This will be available everywhere.

Ex 1:

This create a global variable for $myColor as green.

You can then use it in a different tag even though it was created in another tag.

The output of the paragraph is now green.

$myColor: red;

h1 {
  $myColor: green !global;
  color: $myColor;
}

p {
  color: $myColor;
}

Nesting

You can nest code within blocks instead of writing the full selector out.

The & writes copies the selector it's nested in.

Ex 1:

Nesting the li inside the .navigation class.

.navigation {
  list-style: none;

  //nested selector
  //Otherwise this would be written as: .navigation li
  li {
    display: inline-block;
    margin-left: 30px;
  }
}

<ul class="navigation">
  <li><a href="#">About us</a></li>
  <li><a href="#">Pricing</a></li>
  <li><a href="#">Contact</a></li>
</ul>

Ex 2: Using the & symbol

The & writes copies the selector it's nested in. Otherwise you have to write .navigation li:first-child.

This & symbol copies the li element.

.navigation {
  list-style: none;

  //nested selector
  li {
    display: inline-block;
    margin-left: 30px;

    //first child
    //The & writes out the code for this selector
    //Equivalent to .navigation li:first-child
    &:first-child {
    margin: 0;
    }
  }
}


<ul class="navigation">
  <li><a href="#">About us</a></li>
  <li><a href="#">Pricing</a></li>
  <li><a href="#">Contact</a></li>
</ul>

Ex 3: Using the & symbol with links

This uses the & symbol with links.

.btn-main {
  This is same as: .btn-main:link
  &:link{
    background-color: $color-secondary;
  }

  This is same as: .btn-main:hover
  &:hover {
    //color function: darken by 15%
    background-color: darken($color-secondary, 15%)
  }
}

Creating a function

Ex 2: function to divide

This creates a function that divides two values.

The values are then passed when the function is called.

Create a function named divide
@function divide($a, $b) {
  @return $a / $b;
}

nav {
  //Use function to set 30px
  //* 1px is needed to add the value
  margin: divide(60, 2) * 1px;
}

Mixins

Overview

Reusable piece of code in separate file. You can then add it anywhere.

Ex 1:

Add clearfix code to a mixin named clearfix.

Use @import to add that code to the <nav> section.

//Adding a mixin for the clearfix>
@mixin clearfix {
  //Adds pseudo element after collapsed element
  //This is equivalent to writing nav::after
  &::after {
  content: "";
  clear: both;
  display: table;
  }
}

nav {
  margin: 30px;
  background-color: $color-primary;

  @include clearfix;>
}

Arguments

You can use arguments in the mixin to specify a value.

Ex 1:

This changes the color depending on which color variable is called in the @include statment.

//Add ($color) argument to mixin
@mixin style-link-text($color) {
  text-decoration: none;
  text-transform: uppercase;
  The $color variable will become what was passend into the @include statment
  color: $color;
}

//This passes the $color-text-dark variable
a:link {
  @include style-link-text($color-text-dark)>;
}

Extends

Opposite of mixin. @extend copies the selector, not the content.

Allows you to write a placeholder and put styles in it and have other selectors EXTEND that placeholder.

Only use with similar related selectors.

Start with % sign.

Put repeated code into the block.

Finally, EXTEND it into another block of code.

Ex 1:

Copies Extends code into another block

// Extends
  %btn-placeholder {
  padding: 10px;
  display: inline-block;
  text-align: center;
}

// new block of code to EXTEND info
.btn-main {
  @extend %btn-placeholder;
}

Compiling Sass locally

Install sass using NPM.

Create a package.json file

This should be created in the home directory of your project.

You are prompted for the following questions:

  1. Accept name or give it a new one.
  2. Version: Accept
  3. Description: Leave blank or enter something
  4. Entry point: Accept or enter the project's main file
  5. test command: Leave blank or enter a command such as Standard.
  6. git repository: Leave blank or enter the git repository URL
  7. keywords: Leave blank or enter keywords others could use when searching for your module.
  8. author:
  9. License: this defaults to ISC - most open-source Node.js projects are MIT
npm init

Install sass

Save it as a dev dependency.

npm install node-sass --save-dev

Write a script in the package.json file to compile sass

Edit the package.json file. Add the compile:sass command to the scripts section.

Then add the input file and output file. (This will overwrite the existing style.css file)

The -w flag watches for changes in the main.scss file. If it changes, it is automatically compiled.

{
  "name": "natours",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "compile:sass": "node-sass sass/main.scss css/style.css -w"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "node-sass": "^5.0.0"
  }
}

Run this command to compile.

Since the -w flag is being used, the script will remain running in your terminal and update whenever you save your main.scss file.

npm run compile:sass

npm install live-server -g
