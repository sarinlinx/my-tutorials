Animations

An animation lets an element gradually change from one style to another.

To use CSS animation, you must first specify some keyframes for the animation.

Keyframes hold what styles the element will have at certain times.

The @keyframes rule

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style at certain times.

To get an animation to work, you must bind the animation to an element.

backface-visibility hack

Sometimes, animations cause a very small glitch (shift) at the end. To fix this add backface-visibility: hidden; to the element that shakes.

Simple example

This creates a square that changes color.

  1. Create an HTML element
  2. Add CSS to it. Give it a default background color, animation-name, and animation-duration.
  3. Create a @keyframes block that's binded to the HTML elment's animation-name.
  4. Use from (0%) and to (100%) to set the time of the changes.
  5. The total time of the animation is set by animation-duration

The animation 

<div id="test-animation"></div>
                  

#test-animation {
  width: 100px;
  height: 100px;
  background-color: red;
  animation-name: example;
  animation-duration: 4s;
}

@keyframes example {
  from {background-color: red;}
  to {background-color: yellow;}
}

Alternate directions

The animation-direction property specifies whether an animation should be played forwards, backwards or in alternate cycles.

The animation-direction property can have the following values:

  • normal - The animation is played as normal (forwards). This is default
  • reverse - The animation is played in reverse direction (backwards)
  • alternate - The animation is played forwards first, then backwards
  • alternate-reverse - The animation is played backwards first, then forwards

This reverses the animation.

Set speed curve of animation

The animation-timing-function property specifies the speed curve of the animation.

The animation-timing-function property can have the following values:

  • ease - Specifies an animation with a slow start, then fast, then end slowly (this is default)
  • linear - Specifies an animation with the same speed from start to end
  • ease-in - Specifies an animation with a slow start
  • ease-out - Specifies an animation with a slow end
  • ease-in-out - Specifies an animation with a slow start and end
  • cubic-bezier(n,n,n,n) - Lets you define your own values in a cubic-bezier function

animation-fill-mode

The animation-fille-mode property specifies the style of the elment either:

  • before it starts
  • or after it ends

Otherwise, the element will immediately revert to its original style when the animation is complete.

The animation-fill-mode property can have the following values:

  • forwards - The element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count)
  • backwards - The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period
  • both - The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions

forwards

The original color of the element is red.

Th first animation sets a color.

Since animation-delay is set, it overrides the original color of red.

When the animation is over, the original position and color display.

Shorthand

The shorthand properties go in this order:

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction
div {
  animation: example 5s linear 2s infinite alternate;
}

Example using translate

transform allows you to animate horizontally.

This moves in from the left with opacity zero.

At 80% is goes too far to the right.

At 100% it moves into it's centered location with opacity of 1.

 #translate-1 {
  display: block;
  font-size: 1.1rem;
  color: #fff;
  /* letter-spacing: 3.5rem; */
  /* Set the animation name and duration */
  animation-name: moveInLeft;
  animation-duration: 1s;
  /* How animation should progress over time */
  animation-timing-function: ease-out;
}

@keyframes moveInLeft {
  /* Start */
  0% {
      opacity: 0;
      /* Animates in X plane */
      transform: translateX(-10rem);
  }
  /* 80% of animate time */
  /* 1rem moves further to the right */
  80% {
      transform: translateX(1rem);
  }
  /* End of animation */
  100% {
      opacity: 1;
      /* 0 is its normal position */
      transform: translate(0);
  }
}
Moving to the right, then back to the left
