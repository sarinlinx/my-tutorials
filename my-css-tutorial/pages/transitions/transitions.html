Transitions

Transition styles have two properties:

  • The first property defines which of the many CSS styles are part of the transition.
  • The second property or parameter defines the duration of the transition in ms or sec.

These can be written shorthand like this:

.classname {
  background-color: green;
  left: 0px
  width: 50px;
  height: 50px;
  border-radius: 0%;

  transition: background-color 3s;
}

You could also write these longhand with two style rules:

.classname {
  background-color: green;
  left: 0px
  width: 50px;
  height: 50px;
  border-radius: 0%;

  transition-property: background-color;
  transition-duration: 3s;
}

Example

Example code

<div></div>

div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s;
}

div:hover {
    width: 300px;
}

Hover over the red block.

Multiple values

You could also specify multiple values to change in one declaration.

.classname {
  background-color: green;
  left: 0px
  width: 50px;
  height: 50px;
  border-radius: 0%;

  transition: background-color 2s,
              left 3s,
              width 3s,
              height 3s,
              border-radius 3s;
}

If a property doesn't have a time value, it changes immediately.

Animatable Properties

Only certain properties can be animated. Essentially, any CSS style that can be expressed in terms of numbers is animatable because a browser can calculate values between the initial and end states.

  • border-width
  • Height
  • bottom
  • opacity
  • left
  • width
  • colorsfont-size
  • background-color

Properties you cannot animate.

  • display
  • font-family
  • position
  • background-image
  • border-style
  • white-space
  • column-span
  • table-layout

Timing

Managing timing

You can add timing with a timing property after the duration.

This specifies the rate at which the transition progresses.

transition: background-color 2s timing;

You could also write them longhand like this.

transition-property: property;
transition-duration: duration;
transition-timing-function: timing;

Timing keywords

Timing has 5 keywords

  • ease - the style changes slowly, speeds up in the middle, and then ends slowly. This is the default timing
  • linear - maintains a constant rate of change throughout the duration.
  • ease-in - starts slow and then speeds up to the end.
  • ease-out - starts fast and then slows down to the end.
  • ease-in-out - starts slower than ease timing, speeds up, and then slows down toward the end.

Creating your own timing

You are not limited to the 5 keyword timing options. You can create your own using a cubic-bezier function.

This function plots a graph that the time will follow.

It's syntax is: transition: property duration cubic-bezier(x1, y1, x2, y2)

example

This examples shows it longhand. You could also write it short like this.

transition: width 2s cubic-bezier(0.1, 0.7, 1.0, 0.1)

<div></div>

div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s;
  transition-timing-function: cubic-bezier(0.1, 0.7, 1.0, 0.1);
  }

  div:hover {
  width:300px;
}

Example of 'cubic-bezier'.

Step transitions

Transitions are incremental steps by default. This allows the transition to appear like it's moving.

You can also manually set the steps of the transition.

There are 4 options

  • step-start - stays in the end state throughout the entire duration of the transition
  • step-end - transition stays in the initial state for the entire duration.
  • steps(n, start) - steps function which breaks the transition into a series of steps, so the transition jumps between the different steps and is not gradual. With the keyword start, the first step occurs at the start of the transition.
  • steps(n, end) - with the keyword end, the last step occurs at the end of the transition.

example

step-start

<div></div>

div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s step-start;
  }

  div:hover {
  width:300px;
}

Example of 'step-start'.

step-end

<div></div>

div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s step-end;
  }

  div:hover {
  width:300px;
}

Example of 'step-end'.

steps(6, start)

<div></div>

Delay

The delay value delays the start of the transition.

Why would you want to delay a transition? This is when there are several styles which are changing and you want them to be applied in sequential order.

Example

<div></div>

div {
  width: 100px;
  height: 100px;
  background: red;

  transition: width 1s ease 0s,
              height 1s ease 1s,
              background 1s ease 2s;
}

.div:hover {
  width: 300px;
  height: 150px;
  background: blue;
}

These values change at different times due to the delay value.

Asymmetric transitions

Symmetric transitions are when the transition goes forward to the end state, then reverses itself as the object returns to its initial state

Asymmetric transitions are when the transition going forward is different than the transition coming back.

To acommplish this, you need to create a transition style for both the initial and end states.

Example

This shows transition styles for the initial state and end state of the object.

Note how transitions are applied 'towards' a state. This means the following:

  • The effect of the end transition is applied first when the object is changing TO the end state.
  • The effect of the initial transition is applied when the object is changing back to the initial state.
<div></div>

/* default initial state with transition
  this is applied only when the original end state is over
  and the transition is coming back to the initial state */

div {
  width: 100px;
  height: 100px;
  background: red;

  transition: width 1s ease 0s,
              height 1s ease 0s,
              background 1s ease 3s;
}

/* add transition to end state that's different from initial state
   this allows the transition back to be different
   this transition is applied first when hovered */
   
.div:hover {
  width: 300px;
  height: 150px;
  background: blue;
  border-radius: 25%;

  transition: border-radius 1s ease-in 0s,
              width 1s ease-out 1s,
              height 1s ease-out 2s,
              background 1s ease-in-out 3s;
}

These values change at different times due to the delay value.
