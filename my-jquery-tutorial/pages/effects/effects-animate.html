Overview

The jQuery animate() method is used to create custom animations.

SYNTAX

$(selector).animate({params},speed,callback);

The required params parameter defines the CSS properties to be animated.

The optional speed parameter specifies the duration of the effect. It can take the following values: "slow", "fast", or milliseconds.

The optional callback parameter is a function to be executed after the animation completes.

Is it possible to manipulate ALL CSS properties with the animate() method?

Yes, almost! However, there is one important thing to remember: all property names must be camel-cased when used with the animate() method: You will need to write paddingLeft instead of padding-left, marginRight instead of margin-right, and so on.

Also, color animation is not included in the core jQuery library. If you want to animate color, you need to download the Color Animations plugin from jQuery.com.

animate()

The following example demonstrates a simple use of the animate() method; it moves a

element to the right, until it has reached a left property of 250px:

<button id="btn1">Start Animation</button>
<p>By default, all HTML elements have a static position, and cannot be moved.</p>
<p>To manipulate the position, you must set the CSS position property of the element to relative, fixed, or absolute.</p>
<div idi="d1style="background:#98bf21;height:100px;width:100px;position:absolute;"></div>

$("button#btn1").click(function() {
  $("div#d1").animate({
      left: '250px'
  });
});

Using Relative Values

It is also possible to define relative values (the value is then relative to the element's current value). This is done by putting += or -= in front of the value:

<button id="btn3">Start Animation</button>
<div id="d3" style="background:#98bf21;height:100px;width:100px;position:absolute;"></div>

$("button#btn3").click(function(){
 $("div#d3").animate({
   left: '250px',
   height: '+=150px',
   width: '+=150px'
 });
});

Queuing animations

By default, jQuery comes with queue functionality for animations.

This means that if you write multiple animate() calls after each other, jQuery creates an "internal" queue with these method calls. Then it runs the animate calls ONE by ONE.

So, if you want to perform different animations after each other, we take advantage of the queue functionality:

<button id="btn5">Start Animation</button>
<div id="d5" style="background:#98bf21;height:100px;width:100px;position:absolute;"></div>

$("button#btn5").click(function(){
  var div = $("div#d5");
  div.animate({height: '300px', opacity: '0.4'}, "slow");
  div.animate({width: '300px', opacity: '0.8'}, "slow");
  div.animate({height: '100px', opacity: '0.4'}, "slow");
  div.animate({width: '100px', opacity: '0.8'}, "slow");
});
